# 事故紀錄與演練

## 事故紀錄

發生真的事故時，用 `2026-07-19-blank-page.md` 的格式（依計畫書第 12 部模板）在這個目錄下新增一份
紀錄——只寫已驗證的事實，推測原因另外標註，不混在一起。

## 每季演練清單

依計畫書 §31，每季至少手動演練一次以下情境，確認故障預防機制真的有效，而不是只在心裡覺得應該
有效：

- [ ] **壞 chunk**：把 `dist/assets/` 裡任一個真實 chunk 檔案改名或刪除，確認前端顯示錯誤畫面
      （`EML-FE-CHUNK`）而不是黑畫面，且一次性自動重試機制正常運作。
- [ ] **壞 manifest**：讓 `/ai/manifest.json` 回傳無效內容，確認 `/cases` 顯示降級訊息而不是空白頁。
- [ ] **Worker 失敗**：模擬 `/ai/tools/*` 其中一個工具拋出例外，確認回傳結構化錯誤（非 500 裸露
      stack trace），且不影響其他工具。
- [ ] **回退演練**：實際跑一次 `docs/deployment.md` 描述的回退流程（Dashboard 或 CLI），確認真的能
      在數分鐘內恢復到上一個健康版本。

每次演練後，把結果（成功/失敗、花費時間、發現的缺口）記在這個檔案下面，當作歷史紀錄。
